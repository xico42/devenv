# Design: `internal/envtemplate` package

**Date:** 2026-03-05
**PRD:** `docs/prds/prd-phase-02-env.md`
**Scope:** Pure library package only. CLI wiring (`devenv worktree env`, `devenv worktree new` env step, `devenv session start --export-env`) is handled by the worktree and session command PRDs, which depend on this package.

---

## Package API

One file (`envtemplate.go`) + one test file (`envtemplate_test.go`). No sub-packages.

```go
package envtemplate

type EnvTemplateContext struct {
    Project      string // project name from config (e.g. "myapp")
    Branch       string // branch name (e.g. "feature")
    WorktreePath string // absolute path to worktree
    SessionName  string // "<project>-<branch>"
}

// Process renders a .env.template string with the given context and returns
// the full .env file content including a generated header.
// source describes where the template came from (e.g. "repo-local", "/path/to/file").
func Process(templateContent, source string, ctx EnvTemplateContext) (string, error)

// DeterministicPort returns a stable port for the given project/branch/name.
// Range: 10000–59999. Same inputs always produce the same port.
func DeterministicPort(project, branch, name string) int

// ParseEnvFile reads a .env file and returns key-value pairs.
// Skips blank lines and lines starting with "#".
// Splits on the first "=" only (values may contain "=").
// Used by session start --export-env.
func ParseEnvFile(path string) (map[string]string, error)
```

---

## Template Execution (`Process`)

Uses `text/template` with a `template.FuncMap` of two functions:

- **`port "name"`** — calls `DeterministicPort(ctx.Project, ctx.Branch, name)`; returns the int (rendered as decimal string by the template engine)
- **`env "VAR" "default"`** — calls `os.Getenv("VAR")`; returns the default if empty or unset; never errors; if no default provided and var is unset, returns `""`

Output format:

```
# Generated by devenv — do not edit (regenerate with: devenv worktree env <project> <branch>)
# Template: <source>

<rendered template content>
```

The header is prepended by `Process` itself using the `source` argument. The `source` label is supplied by the caller (e.g. `"repo-local"` or the absolute config path).

## Port Algorithm (`DeterministicPort`)

```go
func DeterministicPort(project, branch, name string) int {
    key := project + "\x00" + branch + "\x00" + name
    h := fnv.New32a()
    h.Write([]byte(key))
    return int(h.Sum32()%50000) + 10000 // range: 10000–59999
}
```

- **Hash:** FNV-1a 32-bit (`hash/fnv`, stdlib — no new deps)
- **Separators:** null bytes prevent `("ab","c","d")` colliding with `("a","bc","d")`
- **Branch:** used as-is (original name, not directory-flattened)
- **Collisions:** no detection or retry; rare across 50,000 slots; user renames the port label to resolve

---

## `.env` Parser (`ParseEnvFile`)

Line-by-line, no external deps:

- Skip blank lines
- Skip lines starting with `#`
- Split each line on the **first** `=` only — values may contain `=`
- No whitespace trimming — strict KEY=VALUE
- Missing file → error (caller decides if the file is optional)

## Testing Strategy

All tests in `envtemplate_test.go`.

### `DeterministicPort`
- Same inputs → same port (idempotency)
- Port in range 10000–59999
- Different name → different port
- Null-byte separation: `("ab","c")` ≠ `("a","bc")`

### `Process`
- All context fields render correctly (`.Project`, `.Branch`, `.WorktreePath`, `.SessionName`)
- `port` function produces value matching `DeterministicPort` directly
- `port` called twice with same name → same value in output
- `env` with set env var → returns var value
- `env` with unset var + default → returns default
- `env` with unset var + no default → returns `""`
- Header comment present in output
- Template syntax error → returns error

### `ParseEnvFile`
- Normal KEY=VALUE pairs parsed correctly
- Comments (`#`) and blank lines skipped
- Value containing `=` parsed correctly
- Missing file → error
